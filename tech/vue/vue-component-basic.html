# Vue组件基础

在Vue中，组件(Component)作为独立可重用的UI模块，是组成页面或应用的基本元素，通常以树状的方式组织起来：

除了UI样式，Vue的组件中根据应用场景的需求，还包含了对应的逻辑部分。

# 事件

事件是子组件向父组件发送数据进行通信的一种途径，主要可以分为两类：

  • 普通事件

父组件通过指令v-on来监听@XXX事件，子组件通过 vm.$emit(eventName, […args]) 的方式触发事件

  • 修饰符事件

v-on指令除了能够监听子组件触发的自定义事件，还可以监听原生DOM事件。 另外，当使用.stop，.prevent等修饰符来修饰普通事件，修饰符可以用于处理原生HTML元素的事件细节。 具体的修饰符和使用方式参见修饰符示例

TIP

$emit方法以数组的方式可以抛给父组件多个参数，父组件中通过展开运算符可以获取到对应参数。 另外还见过一种骚操作就是$emit出去一个callback函数给父组件，从而父组件可以执行callback函数把返回值传给父组件。

# 插槽

在Vue中，一个组件可以通过自定义的props来接收外部传来的数据，以及对相应数据的处理和使用方式。

为了提高组件的可扩展性，适应各种未知的业务场景， Vue还提供了插槽机制，可以让外部灵活地传递UI内容给子组件。 关于插槽的具体的语法规则介绍请参见官方文档 插槽主要可以分为两类：

  • 普通插槽

普通插槽和具名插槽的作用都是让父组件能够将UI内容传递给子组件进行渲染， 但是需要注意的是：

父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。

也就是说，插槽提供的UI内容是在父级渲染的，因而无法使用子组件中的数据和属性的。 为了让插槽所提供的内容能够访问到子组件中的数据，我们需要使用作用域插槽。

  • 作用域插槽

作用域插槽为我们提供了在插槽内容中访问子组件中数据的能力。 在子组件中定义插槽的时候，我们可以将指定属性绑定在插槽元素上，这被称为插槽Prop：

<span>
  <slot v-bind:user="user">
    {{ user.lastName }}
  </slot>
</span>

而在父作用域使用的时候，则通过以下方式引用即可：

<current-user>
  <template v-slot:default="slotProps">
    {{ slotProps.user.firstName }}
  </template>
</current-user>

从上面可以看到，在Vue 2.6之后，具名插槽和作用域插槽都被统一成了v-slot指令，语法变化比较大，具体请参考废弃了的语法
