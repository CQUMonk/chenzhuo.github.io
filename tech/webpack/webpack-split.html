# Code Splitting in Webpack

问题：随着业务的开发与迭代， 我们页面中所引用的JS文件体积也越来越大， 非常不利于浏览器缓存和页面响应速度。

为了解决这个问题， 通常会在工程构建中采用代码划分（Code Splitting）的方式来解决：

  • 将公共模块代码与业务模块分离
  • 实现业务模块的按需加载

本文对 Webpack 进行工程构建过程中的代码拆分操作进行了总结和整理。

# 构建流程

在讨论代码划分之前，首先简单概述一下Webpack进行工程构建的整个过程。

在Webpack的Config文件中， Entry配置指定了整个工程的入口Module。 当Webpack启动并开始执行构建时， 将以Entry所指定的Module为起点， 遍历查找它所依赖（引用）的所有Module，并通过相应Loader进行转换和解析。 解析Module时候又会得到该Module所依赖的Module， 从入口模块开始，整个过程层层递归。 每个Entry及其所依赖的Module都会被划分在同一个Chunk中， 并最终输出到对应的Output文件中。

Entry的类型可以为String， Array， Object或者它们之间的组合，其中只有Object类型会输出多个Chunk.

从流程中不难看出，我们可以按照自己的需求将不同的Module拆分到指定的Chunk，并输出到不同构建结果中。
