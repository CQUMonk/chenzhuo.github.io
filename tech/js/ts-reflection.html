# TypeScript 中的装饰器与反射机制

通常情况下，反射机制是一门编程语言在运行时进行元数据查询和修改的自举能力。

# 注解与装饰器

虽然注解与装饰器非常相似，二者都可以为我们的代码声明添加元数据，但还是略有不同：

通过注解，开发者可以在进行类定义的同时添加元数信息，但是无法控制如何向代码添加元数据，而通过装饰器则开发者可以自己定义元数据是如何被加入到指定类中。

# Decorators in TypeScript

在TypeScript中共有四种类型的装饰器：

  • ClassDecorator 类装饰器
  • PropertyDecorator 属性装饰器
  • MethodDecorator 方法装饰器
  • ParameterDecorator 参数装饰器

它们分别被用于装饰：类，类属性，方法和参数，下面总结一下各种装饰器的使用方式。

# MethodDecorator

方法装饰器的定义如下：

declare type MethodDecorator = <T>(target: Object, propertyKey: string | symbol, descriptor: TypedPropertyDescriptor<T>) => TypedPropertyDescriptor<T> | void;

方法装饰器是一个函数，它接收三个入参：被装饰的方法所在的类对象（原型对象）target，方法名称/唯一标识propertyKey，方法的对象描述符descriptor；

通过方法装饰器，如果我们能在运行时方法的一些元数据，就可以来做一些有意思的事情。

根据定义，我们定义了一个测试类MyClass和一个方法装饰器infos，用来打印方法在执行过程中的一些信息：

function infos(target: Object, key: string, descriptor: any) {
  const methodObj = descriptor.value;
  descriptor.value = function(...args: Array<any>) {
    const argumentStr = args.map(ar => JSON.stringify(ar)).join();
    let result = methodObj.apply(this, args);
    console.log(`调用方法：${key}, 入参：${argumentStr}`);
    console.log(`返回结果:${JSON.stringify(result)}`);
    return result;
  };
  return descriptor;
}

class MyClass {
  @infos
  add(a:number,b:number){
    return a+b;
  }
  @infos
  bb(str:string){
    console.log("bb"+str);
  }
}
